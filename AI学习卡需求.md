## AI学习卡
### 技术栈
1. 使用git进行代码管理，每个大功能模块完成后都进行git提交
2. 前端框架（适配PC+移动端）：Next.js 15 (App Router)
3. UI 组件库：Shadcn UI + Tailwind CSS
4. 前端流式适配：Vercel AI SDK
5. 数据库：Supabase (pgvector) + Drizzle ORM
#### 相关技术要求：
* **响应式设计**：在手机端（sm）容器宽度 100%，在桌面端（lg）限制最大宽度。    
- **交互优化**：判断题按钮在手机上要足够大，方便手指点击。    
- **流式反馈**：使用 Vercel AI SDK 的 `useChat` 展示 AI 反馈，确保长文本在小屏幕上自动换行且可滚动。”
- **Next.js 15**利用 **Server Actions** 处理 Supabase 数据的增删改，减少前端代码量，提升手机端加载速度。注意：在 Vercel 部署时，确保你的 API 路由使用 `Edge Runtime`，这样 AI 的流式响应在手机端会感觉更快（延迟更低）
- **Drizzle + Supabase**在开发 RAG 功能时，你可以直接在 Drizzle 中定义 `vector` 类型。建议在数据库中增加一个 `device_type` 字段。
### 角色
1. 教师
2. 学生
#### 功能
1. 教师信息维护（教师操作）
	* 教师注册，并维护信息
	* 教师信息：学院、专业、教师姓名、姓名、手机号码
2. 大语言模型维护（教师操作）
	* 教师可以管理多个大语言模型基本信息（模型显示名、模型名、base_url、api_key）
	* 教师可以选择一个应用的默认模型，后续所有题目的判分和个性化批改提示都使用该默认模型进行调用
3. 课程信息维护（教师操作）
	- 仅新建该课的老师可进行增删改查 
	- 课程信息：学年、学期、课程名称、人数、教学班组成、教师	- 
4. 学生信息维护（教师操作）
	* 仅新建该课的老师可进行增删改查 
	* 须先选择某个建好的课程，然后通过Excel文件一键导入学生信息
	* 如果学生已存在在学生表中（学号为标识符，以前课导入过），则不用导入该学生信息，并将学生选修（关联）到该课上
	* 如果学生表中没有该学生，则自动导入学生表中，并将学生选修（绑定）到该课上
	* 学生信息：学号、姓名、性别、学院、年级、专业、班级、手机号码、电子邮箱、是否重修
	* 可进行增删改查
5. 教学课堂信息维护（教师操作）
	* 仅新建该课的老师可进行增删改查 
	* 须先选择某个建好的课程，和课程信息发生关联
	* 教学课堂信息：课堂日期（必填），课堂时间（必填），课堂名称，课堂教室，授课教师（默认教师名），备注
	* 可进行增删改查
6. AI学习卡信息维护（教师操作）
	* 要求界面简洁美观，层次分明
	* 仅新建该课的老师可进行增删改查 
	* 在课堂列表中，选择某个课堂，进入AI学习卡管理界面，和课堂信息发生关联，可对该课堂关联的AI学习卡进行增删改查
	* AI学习卡信息（分成不同类型的题型，不同类型的题型具有不同的信息维护，要求操作界面直观简易）。注：每道题都有提交按钮，可以单独提交
		1. 学习卡名称（默认课堂名）
		2. 自我评测打分，包含以下3部分
			* 学习内容名称
			* 自评学习程度：五星值（0-5颗星），点击星值后自动提交
			* 自评学习评语：一个文本框（提示信息：本部分学习的感悟、问题及建议，输入时提示词清空），自评学习评语需点击提交按钮后提交
		3.  多选题（分值，题干、自定义数量的选项、标准答案，个性化批改提示）
			* 多选题默认个性化批改提示根据内置模板生成（教师可以编辑该批改提示，每个题目有每个题目的批改提示）
		4. 填空题（分值，题干、题干中可包含的若干个空格、每个空格的标准答案，个性化批改提示）
			* 填空题默认个性化批改提示根据内置模板生成（教师可以编辑该批改提示，每个题目有每个题目的批改提示）
		5. 简述题（分值，题干、待输入答案（支持markdown输入）、标准答案、个性化批改提示）
			* 简述题默认个性化批改提示根据内置模板生成（教师可以编辑该批改提示，每个题目有每个题目的批改提示）
		6. 课堂总分（100分）
	* 教师可进行AI学习卡**发放**(教师在学习卡列表界面操作，之后学习卡状态为**发放**)
		* 学习卡发放后无法进行修改
		* 发放后进入学习卡内容分析界面
	* AI学习卡作答
		* 发放后，所有选课的同学都可以在自己的课堂列表（以课程进行分类）中看到该学习卡
		* 点卡答题卡后，学生可以对每道题目进行作答，但看不到标准答案，每道题都有提交按钮，可以单独提交
		* 填空题的打分根据填空题打分模板构建LLM提示输入，然后从大语言模型返回一个整数数值
		* 简述题的打分根据以简述题打分模板构建LLM提示输入，然后从大语言模型返回一个整数数值			
		* 学生点击提交后不能继续修改，提交后应用大语言模型自动批改多选题、填空题和简述题
	* AI答题卡分析(教师在学习卡列表界面操作后进入分析界面）
		* 显示答题卡所有的题干，但无法看到答案，可以看到每个题目后有**分析**按钮，点击该按钮后
			* 多选题：展示题目信息，标准答案和并图形化表示每个选项的正确选择百分比
			* 填空题：展示题目信息，标准答案和学生答案构成的词云
			* 简述题：展示题目信息，标准答案和学生答案构成的循环弹幕
